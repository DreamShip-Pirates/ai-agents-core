# Skill: Clean Code Linter

**Purpose**: Enforce readability and maintainability standards.

## Rules
- Functions must be shorter than 30 lines.
- Variable names must be descriptive (no `a`, `b`, `tmp`).
- Every function must have a JSDoc block.
- No deeply nested conditionals (max 3 levels).

## Usage
Run before every commit to ensure "Clean Code" principles.
