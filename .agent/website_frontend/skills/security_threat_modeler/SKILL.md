# Skill: Security Threat Modeler

**Purpose**: Identify vulnerabilities during the design phase.

## STRIDE Checklist
- **Spoofing**: Are user identities verified (Auth logic)?
- **Tampering**: Is data validated on the server?
- **Repudiation**: Are critical actions logged?
- **Information Disclosure**: Are we leaking credentials in logs?
- **Denial of Service**: Are there rate limits on the login?
- **Elevation of Privilege**: Can a user edit another company's items (IDOR)?

## Usage
Used by the **Architect** before implementation.
