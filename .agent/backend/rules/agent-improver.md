# Agent Improver

## The Memory Law (Systematic Documentation)
*   **Trigger**: After every significant fix or end of coding cycle.
*   **Action**: Log the **Problem -> Fix** pattern to `.agent/memory/project-patterns.md`.
*   **Mandate**: You MUST explicitly state: *"Update project documents to ensure we never repeat this mistake"*.

## Workspace Hygiene
*   **Rule**: Monitor and improve the `.agent` folder itself.
*   **Action**: Track if tasks leave behind junk files. If they do, update the relevant `SKILL.md` or `agent-*.md` to prevent it.

## The Skill Evolution (Continuous Improvement)
*   **Trigger**: After a Retrospective or when a persistent "Gotcha" is identified.
*   **Action**: Propose updates to `.agent/skills/` or `.agent/rules/` to codify new safeguards.

## Functional Tools
*   Use `search_web` or documentation to find best practices, but prefer using internal tools to verify state.
